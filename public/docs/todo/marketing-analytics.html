I. Core Financial Metrics (The "North Star")

Direct mapping to P&L health. These metrics answer: "Are we growing efficiently?"

Metric Definition Technical Logic Target
CAC Payback Months to break even on acquisition cost. CAC / (ARPU * Gross_Margin_%) < 12 Months
LTV:CAC Lifetime efficiency. (Avg_Lifespan * ARPU) / CAC > 3:1
Net Churn Revenue lost minus expansion revenue. (Churned_MRR - Expansion_MRR) / Start_MRR < 0% (Negative Churn)

Implementation: CAC Payback Analysis

This query calculates the rolling 3-month payback period, dynamically adjusting for seasonal margin fluctuations.

WITH margin_data AS (
    -- Calculate actual gross margin per customer (Revenue - COGS)
    SELECT 
        customer_id,
        AVG(total_amount - cost_of_goods) as monthly_gross_margin
    FROM orders 
    WHERE status = 'completed' AND created_at >= DATE('now', '-90 days')
    GROUP BY customer_id
),
acquisition_spend AS (
    -- Blended CAC (Ad Spend + Marketing Salaries + Tools)
    SELECT 
        SUM(amount) / (SELECT COUNT(*) FROM customers WHERE created_at >= DATE('now', '-90 days')) as blended_cac
    FROM expenses 
    WHERE category = 'marketing' AND incurred_at >= DATE('now', '-90 days')
)
SELECT 
    ROUND(ac.blended_cac / md.monthly_gross_margin, 1) as payback_months,
    ac.blended_cac,
    md.monthly_gross_margin
FROM margin_data md, acquisition_spend ac
LIMIT 1;

II. Acquisition & Attribution (The "Engine")

Moving beyond "Last Click" to understand the full journey.

Attribution Models

  • Linear: Equal credit to all touchpoints.
  • Time Decay: Actions closer to conversion get more credit.
  • U-Shaped (Recommended): 40% First Touch, 40% Lead Creation, 20% Middle.

Implementation: U-Shaped Multi-Touch SQL

WITH journey AS (
    SELECT 
        user_id,
        source,
        timestamp,
        ROW_NUMBER() OVER(PARTITION BY user_id ORDER BY timestamp ASC) as touch_rank,
        COUNT(*) OVER(PARTITION BY user_id) as total_touches
    FROM marketing_touches
)
SELECT 
    source,
    SUM(
        CASE 
            WHEN touch_rank = 1 THEN 0.4 -- First Touch
            WHEN touch_rank = total_touches THEN 0.4 -- Lead Creation/Conversion
            ELSE 0.2 / NULLIF(total_touches - 2, 0) -- Middle Touches
        END * 100 -- Assuming $100 value per conversion
    ) as attributed_revenue
FROM journey
GROUP BY source
ORDER BY attributed_revenue DESC;

III. Retention & Cohorts (The "Leaky Bucket")

Visualize user lifespan effectiveness.

Implementation: Logarithmic Cohort Matrix

Tracks retention % at Month 1, 3, 6, and 12.

SELECT 
    strftime('%Y-%m', created_at) as cohort_month,
    COUNT(DISTINCT user_id) as total_users,
    SUM(CASE WHEN lifespan_months >= 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as month_1_retention,
    SUM(CASE WHEN lifespan_months >= 3 THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as month_3_retention,
    SUM(CASE WHEN lifespan_months >= 12 THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as month_12_retention
FROM user_stats
GROUP BY 1
ORDER BY 1 DESC;

IV. Organic Growth & Virality (The "Flywheel")

Measuring the self-perpetuating engine of the business.

1. K-Factor (Viral Coefficient)

Formula: Invites per User (i) * Conversion Rate of Invites (c). If K > 1, growth is exponential. 

WITH invites AS (
    SELECT 
        referrer_id, 
        COUNT(*) as invites_sent,
        SUM(CASE WHEN status = 'accepted' THEN 1 ELSE 0 END) as invites_accepted
    FROM referrals
    WHERE created_at >= DATE('now', '-30 days')
    GROUP BY referrer_id
)
SELECT 
    AVG(invites_sent) as i_value,
    SUM(invites_accepted) * 1.0 / SUM(invites_sent) as c_value,
    (AVG(invites_sent) * (SUM(invites_accepted) * 1.0 / SUM(invites_sent))) as k_factor
FROM invites;

2. Content Performance Matrix

Identify "High Traffic / Low Conversion" pages (Optimization Opportunities) vs "High Conversion / Low Traffic" (Promotion Opportunities).

SELECT 
    page_path,
    COUNT(*) as page_views,
    SUM(is_conversion) * 100.0 / COUNT(*) as conversion_rate,
    CASE 
        WHEN COUNT(*) > 1000 AND (SUM(is_conversion) / COUNT(*)) < 0.01 THEN 'Fix Leak'
        WHEN COUNT(*) < 500 AND (SUM(is_conversion) / COUNT(*)) > 0.05 THEN 'Promote Hard'
        ELSE 'Maintain'
    END as strategy_action
FROM page_analytics
GROUP BY page_path
ORDER BY page_views DESC;

V. Driftless Harvest Specifics (The "Yield")

Custom domain logic for local food subscription dynamics.

1. "Box Ecology" (Swap Intelligence)

Understanding product demand via Market Basket Analysis on swaps.

SELECT 
    original_item_id,
    swapped_for_item_id,
    COUNT(*) as swap_frequency,
    -- Calculate "Discontent Score" (Likelihood to be removed)
    (COUNT(*) * 100.0 / (SELECT COUNT(*) FROM box_items WHERE item_id = original_item_id)) as rejection_rate
FROM subscription_swaps
GROUP BY original_item_id, swapped_for_item_id
ORDER BY swap_frequency DESC
LIMIT 10;

2. Sustainability Index (Food Miles)

Marketing metric: "Miles Saved vs Conventional Supply Chain". Assumes conventional average distance is 1500 miles.

WITH delivery_audit AS (
    SELECT 
        d.id,
        d.weight_lbs,
        -- Calculate distance from Farm (origin) to Hub (destination)
        -- Using Haversine formula logic or pre-calculated fields
        (farm.distance_to_hub_miles) as actual_miles,
        1500 as conventional_benchmark_miles
    FROM deliveries d
    JOIN farms farm ON d.farm_id = farm.id
)
SELECT 
    SUM(weight_lbs) as total_lbs_moved,
    SUM(weight_lbs * (conventional_benchmark_miles - actual_miles)) as carbon_miles_saved
FROM delivery_audit;
