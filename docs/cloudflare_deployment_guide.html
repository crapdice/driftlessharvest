Cloudflare R2 Deployment Guide

A step-by-step guide to setting up image management for Harvest Admin

1 Initial App Setup

Dependencies

Ensure you have Node.js installed, then run the following in the root directory:

npm install

Environment Configuration

Create a .env file in the root and add your required secrets:

JWT_SECRET=your_super_secret_key
STRIPE_SECRET_KEY=sk_test_...
STRIPE_PUBLISHABLE_KEY=pk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

2 Account ID & Bucket Creation

Find Your Account ID

On the Cloudflare Home dashboard, look at the right-hand sidebar (you might need to scroll down). You will see a box labeled Account ID. Copy this string; you'll need it for the Harvest Admin settings.

Create Your Storage Bucket

Click R2 in the left-hand sidebar navigation.

Click the blue "Create bucket" button.

Name your bucket (e.g., harvest-images) and click Create bucket.

3 Public Access & CORS Critical

Enable Public Development URL

In your new bucket's dashboard, click the Settings tab.

Scroll down to Public Development URL and click Enable. A confirmation box will appear; click Confirm.

Copy the URL that appears (e.g., https://pub-xxx.r2.dev). This is your Public R2 URL.

Configure CORS Policy

Still in the Settings tab, scroll to CORS Policy and click Edit CORS Policy.

Paste the following JSON exactly as shown:

[
  {
    "AllowedOrigins": ["http://localhost:3000"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE", "HEAD"],
    "AllowedHeaders": ["*"],
    "ExposeHeaders": []
  }
]

4 Generate R2 API Keys

This is the most critical step to allow the Harvest server to talk to Cloudflare.

Navigate to Tokens

Click R2 in the main left-hand sidebar to go back to the R2 Overview.

On the right side of the page, click the link "Manage R2 API Tokens".

Configure the Token

Click the "Create API token" button.

  1. Token name: Enter Harvest Admin Uploads.
  2. Permissions: Select Object Read & Write.
  3. Bucket scope: Choose Specific buckets and select your harvest-images bucket from the list.
  4. TTL (Time to Live): Set this to Forever so your keys don't expire unexpectedly.

Click the "Create API Token" button at the bottom.

Copy Your Credentials

IMPORTANT: These keys are only shown once. Copy them now!

You need to copy two specific values for the Harvest Admin:

  • Access Key ID
  • Secret Access Key

5 Configure Harvest Admin

Update API Keys

Open your Harvest Admin dashboard, go to Configuration > API Keys.

Fill in the Cloudflare fields:

  • Account ID: Found on your Cloudflare dashboard sidebar.
  • R2 Bucket Name: The name you gave your bucket.
  • R2 Access Key ID & Secret: The keys you just generated.
  • Public R2 URL: The .r2.dev URL (or custom domain).

Verify

Open a Product or Template modal and drag an image into the preview area. It should show a "Success" toast and display the preview!

Security Tip: Never commit your config.json or .env files. They contain sensitive keys that should stay local or in your production secrets manager.
